“他们其实真的不想被人发现。”生成式 AI 领域的疯狂竞争意味着就连那些实力最强大的玩家也在抄近路。
TikTok 的推荐算法让字节跳动成为全球舞台的 AI 领导者。
据 THE VERGE 报道，字节跳动如今在生成式 AI 竞赛中处于落伍的境地，以至于它一直在秘密使用 OpenAI 的技术开发自家的大语言模型（ LLM ）。
在 AI 圈子，这种做法通常被认为是一种冒失的行为。
这也直接违反了 OpenAI 的服务条款；其服务条款规定，OpenAI 的模型输出不能用于“开发任何与我们的产品和服务相竞争的 AI 模型”，微软奉行同样的政策。如今字节跳动正通过微软购买 OpenAI 的访问/使用权。
不过，The Verge 记者看到的字节跳动内部文件证实：字节跳动在开发的几乎每个阶段都依赖 OpenAI API 以开发其代号为 Project Seed 的 LLM，包括用于训练和评估模型。
参与其中的员工很清楚其中的含义；The Verge 的记者在字节跳动的内部员工交流平台飞书（ Lark ）上看到过一些对话，讨论如何通过“数据脱敏”来“洗白”证据。滥用现象如此猖獗，以至于 Project Seed 的员工经常达到访问/使用 API 的上限。
在发现字节跳动使用 GPT 训练自己的 AI 模型后，OpenAI 已暂停了该公司的帐户。
在 The Verge 的报道发表后，OpenAI 发言人 Niko Felix 发表了如下声明，确认字节跳动的帐户现已被暂停：
所有API客户必须遵守我们的使用政策，以确保我们的技术被用于正当的用途。虽然字节跳动使用我们的 API 很少，但我们已经暂停了其帐户，同时我们开展进一步的调查。如果我们发现字节跳动的使用不符合这些政策，我们将要求对方做出必要的改变，或者终止其帐户。
正如此前报道的那样，字节跳动使用 GPT 大部分是通过微软的 Azure平台进行的，而不是直接通过 OpenAI。
The Verge 已经询问微软是否会效仿 OpenAI 的做法，同样暂停字节跳动的访问权限。
字节跳动发言人 Jodi Seth 回应称，GPT 生成的数据在“种子项目”的早期开发中用于注释模型，并在今年年中前后已从字节跳动中清除，具体来说已从训练数据中删除。Seth 在声明中说：“字节跳动已经从微软处获得了使用 GPT API 的许可。我们使用GPT 来驱动非中国市场的产品和功能，但使用我们自己开发的模型来驱动豆包，豆包只在中国有售。”
